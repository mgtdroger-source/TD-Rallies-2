Schedule — Route, Timings and Controls

Overview

The Schedule page is used to build the rally route and calculate timings. You add rows of instructions and controls as required. Distance and speed are set for each row of the schedule. As controls are added to the schedule, they are automatically added to the controls page based on the selected rally day.

A rally day can include a second route option. Refer to Route options below.

The page has a Normal/Compact button. Toggling this changes the page size.

Key steps

  • Build the route using instruction and control rows.
  • Define distances and speeds, times are calculated automatically.
  • Control what is to be printed, using tick boxes and colour rules, red is not printed.
  • Manage Speed Groups (SG) by comparing them side by side.
  • Add and manage Added Time adjustments, examples, traffic lights, tea and lunch stops.
  • Select Timeline to review your rally schedule and the interaction of speed groups.
  • Save, Load, Rally projects.
  • Print schedules based on day, route and speed group.

Rows

To add rows to the Shedule table select ADD ROW a selection menu will be presented.

Row types:

  • Instruction — normal route instruction.
  • Instruction rows - 5 off
  • STC — Self Time Control.
  • Marshal — marshal control point.
  • Open Control — entrants can arrive up to 10 minutes early without penalties, there may also be additional rules.
  • Added Time — applies a time delay adjustment, per speed group.
  • Start STC — resets the timing baseline. Entrants record their start time at the control, this time is then used to calculate later STC and Marshal timings.

Font colour and print protocol

To clearly indicate editability and that printed. Colour rules:

  • Blue text — auto-calculated and none editable printed details.
  • Red text — will not appear in print, including Marshal rows.
  • Black text — normal editable and printed content.

Notes:

  • Red text is automatically excluded from a print output.
  • Colour feedback updates immediately as options change.
  • For dist. and speed cells the up and down keyboard arrows can be used to select cells.
  • Selecting Enter after editing a distance or speed cell automatically selects the next cell below ready for data entry.

Tick boxes for printing options

To control whether specific values appear in a printed output.

  • DIST and TIME fields include a tick box.
  • When ticked, the value turns red and not printed.
  • Note: Some row types automatically suppress details from print.

Pristine state (orange warning)

Indicates values that have not yet been edited or confirmed.

  • Newly added row the DIST and SPEED values present an orange ring.
  • The ring clears once the value is edited or confirmed.
  • Pristine is a visual warning only.
  • It does not affect calculations or printing.

Distance validation

Warning of an invalid distance sequence.

  • If a distance goes backwards, the field is flagged by a red border.
  • This highlights a route distance error.

Note: Calculations continue, but the error should be corrected before use.

Speed validation (print clarity)

Speeds are only printed in a following row when the value changes. This is applied automatically.

Gear icon in the first column

When selected, a menu is presented offering a number of insertion options. Inserts are placed in the row below the selected row. There is also a delete option.

Turn column and print selection

Turn instructions can optionally appear in the printed output. For those that prefer to write turnings as part of an instruction text, the Turn column can be hidden.

  • The TURN column has a global print toggle, click “Turn” in the column header it will turn red
  • When disabled, the TURN column is hidden from print
  • Visual feedback shows the column details in red; they will not be printed.

Speed Group (SG) Cards

To show timing and speed information per speed group.

In the button panel above the main table is a listing of the 4 SGs and the maximum speeds of that group. If a group has been allocated (Admin) then the group will include a selection tick box, when selected that SG card will be presented in the schedule table. The cards can be freely selected and deselected, data entered into them being retained when closed.

  • SG cards appear within the Instruction column when selected in the control panel.
  • Each lane shows time and speed for that group, adjust speeds for each group as required.
  • When time of the page row is excluded from print, tick box selected, the times of the SG lanes are automatically excluded.
  • SG's are only provided when setup in the Admin page.

Open Control row

At an open control an entrant can arrive at the control up to 10 minutes (600 seconds) early without penalty, seconds earlier than 600 will be scored as well as any seconds late.

Added Time row

An added row enables the introduction of added time to the next instruction, times added are SG independent. On screen an added time row is taller but in print its row height is the same as any other row.

Added times are included to compensate for delays at traffic lights and time provided for breaks in the schedule, such as lunch.

Added times are applied to each SG individually.

Optionally the added time row can be selected not to print, "NO PRINT", in which case you can use an instruction row above, its dist. being set to the same as that of the added time row, then write your own details. The added row timings will still be applied to the row below it.

Use case

Such timings can be used to give slower speed groups a head start to avoid faster groups needing to overtake in a section where it might be difficult to do so, for example after a lunch break.

Tip: Use Timeline to aid appreciation of groups as they proceed along the route.

Start STC row

At a Start STC the entrant records their elapsed rally time since leaving the rally start. The schedule row indicates time 00:00:00. After a Start STC entrant's still record later STCs based on their rally time from the start of the rally.

Route options

Route options add a second schedule to a rally day.

To enable this feature, Routes in Admin must be set to 2. When selected, an additional Route selector is added to the Schedule page.

Each route is planned separately but must include the same number of controls. A Clone option is provided to copy Route 1 to Route 2, avoiding the need to build Route 2 from scratch.

Controls are divided between routes, as illustrated.

The split route STCs 2 and 4, can be at any location with there own identity within the route sub sections.

Marshal controls cannot be split between routes, as they are designed to record times for all entrants.

Route handling in Controls is automatic. ETA calculations are based on each entrant’s route and speed group.

Timeline (refer below) has two modes when routes are used.

Route 1 view shows Route 1 only.

Route 2 view combines both routes, enabling review of how all entrant groups interact across the rally.

Save and Load

Save and load are common to all pages, selecting save will store a backup of your work in the workspace folder based on the rally title.

Load when selected will present a file picker to enable the selection of which saved rally file is to be loaded, it must be emphasized that loading will overwrite your current work. Note: a warning is shown before loading a file.

Day selection

Purpose: To select which rally day is being viewed and edited in schedule, when you select an alternate day the Controls page for that day will be automatically selected, likewise the same applies selecting a day in Controls, the schedule for that day will be presented. On a rally day the pages for that day are automatically selected on opening the program.

How to use:

  • Use the Day selector at the top of the page.
  • Select the required day.
  • The visible cards and data update immediately.

Print options

When print is selected options are presented, select the day to be printed and for which speed group. The print dialog offering options for portrait or landscape formats. The layout of the printed pages is predefined. Note: Red (non-print) values are omitted automatically.

Timeline

A tool used to review cars and the different speed groups as they proceed along the rally route and review where faster groups pass slower ones.

There are two variants of Timeline, the first for Route 1, the second when reviewing Route 2. Timeline now combines both routes into one.

When the timeline card is opened there are option buttons to select how it is presented on the page.

  • Dock, the card is positioned in the table on the right, the edge of the card can be moved left and right as required.
  • Float, the card can be freely positioned on the page.
  • Normal/Compact, change the size of the card.

Card controls

Start delays section:

Group identifies each of the assigned SG and the entries for that (group).

Delays (min): At the start of the rally added delays between SG can be applied, giving slower groups a head start to avoid faster groups having to overtake when road conditions are less than ideal. Use the “Apply live delays” selector to turn delays on and off.

Note: If the rally day has more than one route start delays and cars can only be configured based on the first route, not both.

Cars: when planning and reviewing the rally using Timeline entries for each SG may not be known, so cars are manually selected as a representation of the rally Cars (entrants) for each group.

Note: In the graphic each SG is illustrated as a train of cars, the dots in the train being the distance between cars, these spacings change dynamically based on speed.

Focus Row

Enables the selection of an instruction/waypoint in the schedule and sets the centre point of the graphic to that point. Listed below is copy of the schedule table instruction text for that waypoint.

TimeLine graphic

Below the graphic are two horizontal scroll bars, the upper one pans the length of the routes whilst the lower one pans 20 minutes either side of the current focused point, when a focus point is changed the time scroll resets to centre. At the bottom of the graphic pane are left and right arrows icons that enable stepping forward and backward along the route.

Waypoints such as STC, Marshal and Added time are colour coded for ease of reference. To the right of a waypoint in front of each train approaching that waypoint are times, A for arrival and associated with an Added time waypoint a D for departure time. Times are shown in rally master time.

Interaction with Admin and Controls – Start delays

If start delay times are to be taken into account, then in Admin when the starting order is slowest groups first an additional option is present in the Control panel to add delay times to the start times of the groups, including delay timings automatically updates the control page ETA timings.
