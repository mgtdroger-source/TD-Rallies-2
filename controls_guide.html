Controls — User Guide

Overview

The Controls page is used on the day of the rally to record marshal controls and STC times, calculate penalties, and publish results. The Controls page reads the list and order of controls from the Schedule page and displays a corresponding set of control cards for each rally day. Controls do not create or change the schedule.

Page details: blue font represents non-editable data. Only Clerk of the Course (COC) penalties and ATA times can be edited.

Results can be entered manually or automatically populated from Google data. Refer to Control Data below.

The page has a Normal/Compact button. Toggling this changes the page size.

Key items

  • A card is shown for each control defined in the Schedule: STCs, Marshals, Open Controls, and Start STCs. Each is numbered as per the schedule sequence.
  • Controls are presented per day using the day selector.
  • Expected times (ETA) are calculated automatically based on the entrant's speed group and route allocation.
  • Actual times (ATA) can be entered manually or automatically using the Controls Data tool.
  • Scores are calculated from the difference between ATA and ETA. Maximum errors are 600.
  • Save and Load use the same workspace as Admin and Schedule.
  • Print results: presents print options, including a summary for multiple rally days.
  • Export, creates an Excel file of the page data which you can then tailor.

Page layout

Purpose: To provide a clear, card-based layout for entering control times and viewing the calculated results.

Description

On the left of the page is the listing of entrant details, including their speed group, route allocation (if more than 1 set - refer Admin) Rally number and lastly their total error score, these fields cannot be edited.

The entrant Rally number when clicked will open a card enabling the results for that entrant to be entered sequentially rather than panning to individual control cards across the page.

To enable comparison of control times to start times, click Total Score, the column below will change to Start Time, toggle again to revert.

To the right of the entrant details is an optional card for the entry of COC penalties, the “offence” and the points to be awarded, to include these penalties in the final score COC must be checked.

Note: When two routes have been selected and a route is missing controls, an Rt. warning is presented in place of ETA timing.

Day selection

Purpose: To select which rally day is being viewed and edited in controls, when you select an alternate day the schedule for that day will be automatically selected, likewise the same applies selecting a day in schedule, the controls for that day will be presented. If a rally day the page will open set to that day.

How to use:

  • Use the Day selector at the top of the page.
  • Select the required day.
  • The visible cards and data update immediately.

Control cards

Purpose: Each card represents one control point and is used to calculate times and scores for each of the entrants based on rally number and speed group.

Card types:

  • STC — Self time control cards as defined in the Schedule
  • Marshal — records arrival times at a fixed control point.
  • Open Control — similar to Marshal but entrants can arrive at the control early without penalties, up to 10 minutes early.
  • Start STC — resets the timing baseline, entrants provide their leaving time from the control. Control ETA timings following a Start STC are automatically calculated based on the Start STC time submitted.

STC, Marshal and Open Control cards

Purpose: To record arrival times (ATA) and calculate penalties, penalties are limited to +/- 600 seconds, the total score is the absolute sum of penalties.

Expected times (ETA) are calculated and cannot be edited (blue font).

ETA time calculations.

  • STC - elapsed time from Schedule based on SG of entrant.
  • Marshals and Open Controls – entrant start time + elapsed time from Schedule based on the SG of entrant.
  • If a Start STC precedes the control: ETA = Start STC time + elapsed time from Schedule.

Note: If a Start STC time has not been entered, then following card ETA times will remain blank.

How to use:

  • Review the calculated ETA.
  • Enter or download ATA times as recorded by the entrants and marshals. Downloaded times automatically populating the cards.
  • Score update automatically.

Starts

Starts when selected will present the starting times for each entrant, enabling any timing checks to be undertaken.

Start STC cards

Purpose: A Start STC resets the rally timing. All following ETA times are calculated from this new starting point. STC and Marshal ETA times being calculated based on the Start STC time submitted.

How to use:

  • Enter or download the Start STC time provided by each entrant.
  • Once entered, downstream ETAs update automatically.
  • Until a Start STC time is entered, expected times for later controls cannot be calculated.
  • Start STC cards do not record penalties themselves.

Note: when a Start STC is incorporated into the schedule, later marshal timings are no longer predictable, so this needs to be appreciated and arrangements made accordingly. Entrants can also be advised that they must leave a Start STC by a given time, thus marshal times can be more easily managed.

Entrant guidelines: At a Start STC the schedule time is 00:00:00, a timing reset in the schedule, the timings to the next instructions being an increment from this fresh start.

The time to be recorded at the Start STC by the entrant is the elapsed time, based on the rally master time since the start of the day. When they reach a next STC they again record their rally master time.

In order to track their progress in terms of time from the Start STC they will need to either start a separate stop watch on leaving the Start STC and track their progress based on the schedule timings or add the schedule times to their Start STC time and use rally master time.

Result Audit

Purpose: A tool that will highlight result errors that exceed the audit value selected, to use the tool first select it (check) and then enter a value. Should a control have excessive errors which could be a result of an unexpected route delay, then that control can be removed from the overall results by unchecking it. All control cards are normally checked. The printed result from an unchecked control will be zero.

Typical workflow

  • Build the route and controls in Schedule.
  • Open Controls on rally day.
  • Select the correct day.
  • Upload your control data and the share links to entrants and marshals.
  • Later review the returns submitted, then download.
  • Enter/download the ATA times provided by each entrant and marshals.
  • Review scores, optionally make use of the Audit tool.
  • Deselect any controls to be excluded from the day’s result.
  • Print results.

Controls data (Google STC & Marshal ATA imports) and (STC Recorder mobile App)

Purpose: STC and Marshal times can be submitted electronically, once submitted the results can be downloaded, the control card ATA times populated ready for review and printings of the day results, thus reducing or eliminating manual entries on the rally day.

How it works: When Controls data is set up and connected, the page can fetch the latest control records from the cloud and populate ATA fields for the relevant controls and entrants.

How to use: Click the Controls data a card will be presented, if the cloud service has been setup in Admin, then "Connected" will be displayed at the top of the card.

If a prior upload of data becomes out of date, changes made since the last upload then the Controls data button will wink, examples: an added entrant, driver name change or control changes.

Card buttons, uploading and sharing links

  • Upload controls data used when all entries and schedule plannings have been finalized, or there has been a change since a last upload.
  • Entrant and Marshal links, either Copy link , Email link or WhatsApp (WA) links to the results forms to be completed.
  • There are also links provided to share the STC Recorder mobile App that can used during the rally, refer to the overview guide for the STC Recorder App summary.
  • To WA links you will need to have WA on your computer, you then send the link to yourself, which you can then share as required. Consider setting up a WA group of entrants, Marshal links will need to be sent separately, or a marshal group.

Card buttons, reviewing and downloading submitted data

  • View form responces – opens the Google form responses page.View mobile responces – opens the Google responses page from mobile submissions.
  • Select Open controls data this opens the Google rally worksheet, select the required tab, review the data awaiting download.
  • Download controls data – only enabled when there is data available for download, select to import data populating the control cards.

Notes: When a download is performed it will overwrite any previously captured data.

Entrant results are checked against their Surname and Rally number, if an error is made, that submission will not be added to the worksheet data tab ready for download, it will be included in the error list tab, select Open controls data to review.

Entrants and Marshals can submit more than one return, the last being used.

Printing of results

Select Print a menu is presented, offering options including overall result from rallly days.

Result data can also be exported for further processing.

Save and Load

Save and load are common to all pages, selecting save will store a backup of your work in the workspace folder based on the rally title.

Load when selected will present a file picker to enable the selection of which saved rally file is to be loaded, it must be emphasized that loading will overwrite your current work. Note: a warning is shown before loading a file.
